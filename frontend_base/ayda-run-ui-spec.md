# Ayda Run — UX/UI Design Specification

## Executive Summary

Ayda Run — Telegram Mini App для управления спортивными сообществами (беговые клубы, хайкинг, вело).

**Решаемые проблемы:**
1. Информационный хаос → структурированное управление мероприятиями
2. Непонятно кто участвует → прозрачность participation

**Дизайн-философия:** Content-first минимализм (адаптация ayda_think)
- Контент занимает 70%+ экрана
- Минимальный header
- Actions внизу
- Никакого visual noise

**Роли:** 2 (organizer / participant)

**Экраны MVP:**
1. Onboarding (3 шага)
2. Home (главный + разбивка по дням)
3. Activity Detail
4. Clubs & Groups List
5. Club/Group Detail
6. Create Activity
7. Create Club
8. Create Group

### Activity Card

```
┌─────────────────────────────────────┐
│ [Название]                    [🏃] │
│ [Время] · [Место]                   │
│ [X км] · ↗[Y м] · ~[Z ч/мин]        │

└─────────────────────────────────────┘
```

### Action Button States

- `[Записаться]` — не записан, есть места
- `[Иду ✓]` — записан (green)
- `[Мест нет]` — full, disabled
- `[Был ✓]` — прошедшее, отмечен (gray)
- `[Пропустил]` — прошедшее, не был (gray)

### Toggle Modes

- **Мои:** где я записан или был
- **Все:** доступные (мои клубы + публичные)

## 13. Key Interactions

### Join Activity
1. Tap card → Activity Detail
2. Tap [Записаться] → instant join
3. Button changes to [Иду ✓ · Отменить]
